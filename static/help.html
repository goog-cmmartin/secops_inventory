Help & Documentation

This guide provides an overview of the different sections of the SecOps Inventory Platform.

Setup Tab

This is the main landing page where you manage your tenants. A tenant is typically a GCP Project that has a SecOps instance. From here, you can:

  • View all discovered or manually added tenants.
  • See whether a tenant has been configured with the necessary API credentials.
  • Click Configure to add or update a tenant's details, including SecOps Customer ID, region, and API keys for services like SOAR and BindPlane.
  • Click Test to verify that the application can successfully connect to the Chronicle API for that tenant.

Chronicle API Tab

This tab provides a raw interface for making ad-hoc requests to the Chronicle API for a selected, configured tenant. It's a useful tool for exploring the API or for debugging.

Audits Tab

This is where you run audits against your tenants. The process is simple:

  1. Select a configured tenant from the dropdown menu.
  2. The list of available audits will be displayed, grouped by category.
  3. Check the boxes for the audits you wish to run.
  4. Click Run Selected Audits. The results will appear in the text box below.
  5. Once you have successful audit runs, you can click Generate Report to create a human-readable summary using the AI model, or Create Schedule to set up recurring audits or reports.

Insights Tab

Insights allow you to correlate data from multiple different audit sources into a single AI-driven analysis. For example, you could create an Insight that looks at both Chronicle rules and SOAR playbooks. You can create, edit, and run Insight templates from this page.

Schedules Tab

The Schedules tab allows you to automate repetitive tasks by creating scheduled jobs. You can define:

  • Schedule Name: A descriptive name for your scheduled job.
  • Tenant: The specific tenant (GCP Project) against which the schedule will run.
  • Schedule Type:
    • Run Audits: Executes a selected set of audits.
    • Generate Report: Generates an AI-summarized report from selected audits.
    • Run Diff: Automatically compares the last two successful runs of selected audits and generates a consolidated diff report, highlighting only configuration changes.
  • Audits to Include: Which audits will be part of this schedule (relevant for all schedule types).
  • Cron Schedule: A standard cron expression to define when the job should run (e.g., "0 1 * * *" for daily at 1 AM).
  • Report Name Format (for Reports and Diffs): A customizable name format for the generated reports.
  • Enable/Disable: Toggle the schedule's active status.

Settings Tab

This tab contains advanced configuration options:

  • LLM Report Customization: Customize the prompt and exclude specific data fields for each audit before it's sent to the AI model for report generation. This allows you to fine-tune the focus of your reports.
  • Custom YL2 Queries: Create and manage your own YARA-L 2.0 queries, which will then appear as runnable audits in the Audits tab.
  • Configurable Audits: Add or modify audit definitions to target different APIs (Chronicle, SOAR, BindPlane) and endpoints.

Reports Tab

This page lists all the AI-generated reports you have created, including regular summaries and diff reports that highlight configuration changes. You can view, copy, or delete them from here.

MCP Assistant Tab

This provides a conversational AI interface for interacting with the application's data. You can ask it questions in natural language (e.g., "how many tenants are configured?") or use slash commands for direct tool execution (e.g., /list_tenants).
